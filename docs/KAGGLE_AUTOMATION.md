# Kaggle automation

The next collaborator may continue b4 after the preflight gate in `docs/HANDOFF.md`. Use single-run tools only. Never submit several jobs together, retry a failed kernel automatically, or expose credentials.

## Components

- `tools/kaggle_submit.py` builds an ignored bundle containing source, experiment configs, requirements, and frozen split CSVs. It generates a private Kaggle script-kernel runner.
- `tools/kaggle_status.py` queries a kernel status.
- `tools/kaggle_collect.py` downloads output and copies splits/results into local ignored artifact directories.
- `tools/kaggle_run_all.py` and `tools/auto_kaggle_queue.sh` are historical queue helpers. Do not use them in the current state because completed or blocked runs could be resubmitted.

## Invariants to review

- Credentials (`kaggle.json`, tokens, `.kaggle/`) never enter the repository or bundle.
- A dataset root directly contains numeric subject folders such as `000/`, `001/`, and `002/`.
- Frozen `splits/*.csv` files are reused; split logic is not changed.
- Generated runners materialize the bundled project and use that project root as subprocess `cwd`.
- Kaggle outputs stay under `/kaggle/working`.
- Only the four fixed run IDs are accepted.
- Runs are sequential, failures stop the queue, and retries are never unbounded.
- No scientific config, model, metric, split, or epoch setting changes as part of infrastructure work.

## Controlled b4 workflow

After the resume unit test and code patch pass, create a dry-run bundle:

```bash
python tools/kaggle_submit.py \
  --run-id b4_mobilenet_softmask \
  --kaggle-user USERNAME \
  --dataset-source OWNER/DATASET_SLUG \
  --dataset-root /kaggle/input/DATASET_SLUG/NESTED_DATASET_ROOT \
  --accelerator NvidiaTeslaT4 \
  --timeout 21600 \
  --dry-run
```

Inspect the bundle against `docs/HANDOFF.md`. Remove `--dry-run` only after every check passes. Then use:

```bash
python tools/kaggle_status.py --run-id b4_mobilenet_softmask --kaggle-user USERNAME
python tools/kaggle_collect.py --run-id b4_mobilenet_softmask --kaggle-user USERNAME
```

If b4 fails, stop. If it succeeds and `test_metrics.json` is verified, repeat the single-run workflow for `ours_iresnet_msff_softmask`.

See `docs/HANDOFF.md` for the collaborator workflow and `AGENTS.md` for mandatory restrictions.
